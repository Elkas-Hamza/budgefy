# 📌 Conception des Diagrammes UML

## 📖 Description du Projet

Ce document présente les diagrammes UML réalisés pour le système de gestion financière.  
Ces diagrammes permettent de comprendre l’architecture, les fonctionnalités et le fonctionnement du système.

---

# 📊 Diagrammes du Système

## 1️⃣ Diagramme de Cas d’Utilisation

Le diagramme de cas d’utilisation montre les fonctionnalités du système ainsi que les interactions entre les acteurs (Utilisateur et Administrateur) et l’application.

Il permet de :
- Décrire les actions possibles dans le système
- Identifier les rôles des acteurs
- Comprendre les fonctionnalités principales

## 2️⃣ Diagramme de Classes

Le diagramme de classes représente la structure du système.

Il contient :
- Les classes principales (Utilisateur, Catégorie, Dépense, Revenu, Tableau de Bord)
- Les attributs de chaque classe
- Les méthodes
- Les relations entre les classes

Ce diagramme décrit l’architecture logique du système.

## 3️⃣ Diagramme de Séquence

Le diagramme de séquence illustre le déroulement des opérations dans le temps.

Il montre :
- La communication entre les objets
- L’ordre des messages
- Le processus d’exécution d’une fonctionnalité (exemple : ajouter une dépense)

Ce diagramme permet de comprendre le fonctionnement dynamique du système.
